Command Line Documentation

FubuDocs has direct support for creating documentation for command line tools build with the LINK TO PROJECT 'FubuCore' command line tooling using the help information already built into your command line tool.

Dumping the Command Usage

The FubuCore command line support includes a command named dump-usages that will export all the command line help information to an Xml file that FubuDocs can parse. The usage of that command looks like the following (taken from the FubuDocs rake script):

FubuDocsRunner.exe dump-usages fubudocs src/FubuWorld.Docs/fubudocs.cli.xml

The first argument fubudocs is the name of the command line executable the way that users will type the commands. Think "git" or "fubudocs" or "bottles."

The second argument is just the path where you want the Xml dump to go. As a naming convention for FubuDocs, name the file [executable name].cli.xml.

Embedding Command Usage

Use the <Command> binding to embed the command line documentation with explanatory prose like:

<Command app="fubudocs" command="run">
....and write some prose here.......
</Command>

The usage above renders this:

fubudocs runRuns a FubuDocs documentation project with Katana

....and write some prose here.......
fubudocs run[-n, --no-bottling] [-u, --url ] [-b, --build ] [-p, --port ] [-a, --application ] [-d, --directory ] [-o, --open] [-w, --watched] [-b, --browser Chrome|Firefox] [-p, --production-mode]
Flags
[-n, --no-bottling]Disables the bottle and code snippet scanning while this command runs
[-u, --url <ur>]If set, the relative starting url at which to open the browser
[-b, --build <build>]If you are running a class library, sets the preference for the profile to load. As in bin/[BuildFlag]. Default is debug
[-p, --port <port>]IP Port. Default is 5500
[-a, --application <application>]Specific name of an IApplicationSource class that builds this application
[-d, --directory <directory>]Overrides the directory that is the physical path of the running fubumvc application
[-o, --open]Start the default browser to the home page of this application
[-w, --watched]Open a 'watched' browser with WebDriver to refresh the page on content or application recycles
[-b, --browser Chrome|Firefox]Browser to use in watched. Default is Firefox because it's more stable.
[-p, --production-mode]Unless this flag is set, the fubumvc application will run in Development mode
